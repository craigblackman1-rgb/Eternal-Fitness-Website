Design System

Eternal Fitness — the visual and component language for eternal-fitness.co.uk. Colours and tokens extracted directly from the Figma design file.

Overview

How the design system is structured, from primitives through to components.

Layer 1: Primitives
Raw colour values (teal-500, rose-400), font sizes, spacing units, shadow definitions. These never carry semantic meaning — they are the building blocks.
Layer 2: Semantics
Purpose-driven tokens that reference primitives. --brand-primary, --text-primary, --surface-card. These encode meaning, not appearance.
Layer 3: Components
Component-specific tokens (--btn-primary-bg) and class-based components (.btn--primary, .card). Built entirely on semantic tokens.
Colour source of truth
Primitives extracted from the Figma design file. Primary Brand = Muted Rose (#C1839F), Secondary = Teal (#087E8B), Highlight = Warm Gold (#C9A96E), Flourish = Sage (#8A9A7B), Text = Charcoal (#3C3C3C), Background = White (#FFFFFF). Interactive accent uses a deeper muted rose (≈#754d63) for functional contrast.

Usage Guidelines

VariantWhen to useWhen NOT to use
Primary (Muted Rose)Main page actions, form submissions, primary CTAsDon't use on muted rose/cream backgrounds — insufficient contrast
Secondary (Teal)Feature CTAs, "Learn more", section navigationDon't use as the dominant page action — reserve primary for that
Highlight (Gold)Special actions, info-driven CTAs, events/promotionsDon't overuse — reserve for one-off emphasis
CTA (Coral)Urgent actions, time-sensitive offers, booking CTAsDon't use alongside primary on the same page — creates confusion
OutlineAlternative to ghost when a visible border is neededDon't use on dark/coloured backgrounds without checking contrast
GhostToolbar actions, close buttons, low-emphasis controlsDon't use for primary page actions
LinkInline text actions, "View all" linksDon't use standalone — looks like regular text
