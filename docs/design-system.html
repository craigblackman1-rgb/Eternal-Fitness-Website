How the hub inherits from branding
All shadcn/ui components (buttons, inputs, selects, badges, cards) reference --primary, --accent, and --ring from globals.css. Updating the design tokens automatically updates the entire hub. The sidebar uses its own --sidebar-* tokens, defined in globals.css, for independent theming.

Avoid These

Common AI-generated design tropes that do not belong in this design system.

Gradient backgrounds
No aggressive purple/violet gradients. Flat colours only — teal, rose, muted, white.
Generic emoji icons
No ✨ 🚀 🎯 as feature icons. Use the SVG icon system or Lucide icons.
Left-border accent cards
No rounded cards with a coloured left border. Use tinted backgrounds or elevated cards.
Invented metrics
No "10× faster" or "99.9% uptime" without a source. Honest placeholders beat fake stats.
Filler copy
No "Feature One / Feature Two" or lorem ipsum. Every section must have real, specific copy.
Icon next to every heading
No automatic icon + heading pairing. Icons are for feature cards and process steps, not section titles.
