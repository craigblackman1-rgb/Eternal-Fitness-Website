Foundation

Colour Palette

Five colours from the Coolors chain, mapped to design roles. The palette uses a deep navy anchor, a cool mint background, a bright cyan accent surface, and a single hot-pink exclamation.

#084B83Deep navy — primary, hero overlays, dark sections
#42BFDDBright cyan — secondary, interactive hover, data highlights
#BBE6E4Soft mint — muted surfaces, light section backgrounds
#F0F6F6Off-white — page background, card contrast base
#FF66B3Hot pink — accent, featured only (use ≤1× per view)
Usage note: The hot pink (#FF66B3) is a very strong accent. Use it for one thing per page — a primary CTA, a testimonial star rating, a single decorative flourish. Using it twice dilutes the signal.
Contrast note: Hot pink #FF66B3 has ~2.9:1 contrast against white — fails AA for normal text, passes for large text (≥18px bold / 24px regular). Always use navy text (#084B83, 4.5:1) on pink backgrounds, never white text.

System

Anti-Patterns

Things the palette discourages — common traps that compromise the system.

✗More than one hot pink accent per view. The moment two pink elements appear on screen, neither reads as special. Choose one — a single CTA, a testimonial star row, or a stat number — and let everything else recede into navy/cyan.
✗White text on hot pink. 2.9:1 fails WCAG AA for normal text. Always use navy text on pink backgrounds.
✗Using mint as a page background. Mint is for alternating sections only. The page background should be #F0F6F6 (off-white) so mint sections visually pop when they appear.
✗Cyan as body text. Cyan at 56% lightness is too light for extended reading. Reserve cyan for data highlights, numerics, and interactive states.
✗Three backgrounds in a row. Never place two non-white sections adjacent. Rhythm: white → accent → white → accent → white.
