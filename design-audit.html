Overview

Three Design Layers That Evolved Independently

The site has accumulated design debt across several rounds of updates. There is no single source of truth for components, spacing, or typography — instead three parallel systems are running side by side and conflicting.

Layer 1 — Homepage

Custom scoped CSS in home.css with its own variables (--ink, --cream, --teal), its own nav, its own button classes, and h2 headings in DM Sans 800.

Layer 2 — Inner marketing pages

Use globals.css utility classes (ef-section, ef-eyebrow, ef-card) plus inline Tailwind. h2 headings inherit the global rule and render in DM Serif Display 400 — a completely different typeface from the homepage.

Layer 3 — Hub back office

shadcn/ui components with Tailwind brand tokens. Largely self-consistent but siloed — uses hardcoded colour values, hardcoded breadcrumb text, and its own SVG conventions.

Design Decisions

Key Calls Before Implementation

Heading font — use DM Serif Display everywhere

The global h2 rule already uses DM Serif Display, and this is the right choice for a healthcare/rehabilitation brand — it reads as warm, trustworthy, and established. The homepage should be brought into line with this, not the other way round. Homepage h2s should switch from DM Sans 800 → DM Serif Display with tight tracking to maintain editorial impact.

CTA label — standardise on "Book a Free Consultation"

"Contact Us" is weaker and less specific than "Book a Free Consultation". Every nav and primary CTA across the site should use the same action-oriented label. This aligns with how the homepage nav already phrases it.

Hub headings — keep DM Sans (this is correct)

The hub is a data-dense back office tool for a trainer, not a consumer-facing page. DM Sans is the right typeface for dashboard headings — it's more legible at smaller sizes in data contexts. This is a deliberate and appropriate divergence that should be preserved.

Implementation Plan

6 Phases — ~9 Hours Total

1

Consolidate the Token System

Add missing tokens to globals.css and make home.css reference them

~1 hour Risk: Low
  • Add --color-ink: #131313, --color-body: #525A61, --color-muted: #7E8088, --color-border-warm: #E4DDD7 to globals.css :root
  • Add ink colour to Tailwind config (for Footer background) and border-warm for border utilities
  • In home.css, replace all local variable declarations with references to globals tokens:
    --ink: var(--color-ink); etc.
  • Update Footer.tsx to use bg-ink instead of bg-[#131313]
  • Result: One source of truth for all brand colours. A single change in globals.css propagates everywhere.
/* Add to globals.css :root */
--color-ink: #131313;
--color-body: #525A61;
--color-muted: #7E8088;
--color-border-warm: #E4DDD7;

/* home.css — replace all local declarations */
.efhome {
  --ink: var(--color-ink);
  --teal: var(--color-teal);
  --rose: var(--color-rose);
  --cream: var(--color-cream);
  --body: var(--color-body);
  --hborder: var(--color-border-warm);
  /* keep --f and --max */
}
2

Unify Typography

Fix the biggest visual inconsistency — heading fonts between homepage and inner pages

~30 min Risk: Low
  • Update .efhome h2.D in home.css to use var(--font-serif) (DM Serif Display), matching the global h2 rule
  • Reduce font-weight from 800 → 700 (serif reads heavier at the same weight value — 700 gives the same impact)
  • Keep the existing size (clamp(34px, 3.4vw, 50px)), line-height (1.06), and letter-spacing (−0.03em) — these maintain the homepage's editorial energy
  • The CTASection.tsx h2 already uses serif — no change needed there
  • Result: Navigate from homepage to About — the heading typeface now matches.
/* home.css — update h2.D */
.efhome h2.D {
  font-family: var(--font-serif); /* ADD */
  font-size: clamp(34px, 3.4vw, 50px);
  font-weight: 700; /* was 800 */
  line-height: 1.06;
  letter-spacing: -.03em;
  color: var(--ink);
}
3

Shared Button Utility Classes

Add reusable .ef-btn-* utilities to globals.css — stop repeating Tailwind strings

~1.5 hours Risk: Low
  • Add .ef-btn base class + .ef-btn-primary, .ef-btn-dark, .ef-btn-outline, .ef-btn-ghost-white variants to globals.css
  • Replace all inline Tailwind button strings across inner pages (About, Pricing, Personal Training, Contact, FAQs) with these classes
  • Rename homepage's existing .btn → .ef-btn and .btn-rs → .ef-btn-primary etc. for naming consistency
  • Hub keeps using shadcn <Button> — appropriate for its data-dense context
  • Result: All marketing page buttons share one definition. Change the pill radius or padding once, it updates everywhere.
/* Add to globals.css @layer utilities */
.ef-btn {
  display: inline-flex; align-items: center; gap: 8px;
  padding: 12px 24px; border-radius: 999px;
  font-size: 14px; font-weight: 600; cursor: pointer;
  transition: all 200ms; border: none;
}
.ef-btn-primary   { background: var(--color-rose); color: #fff; }
.ef-btn-primary:hover { opacity: .88; }
.ef-btn-dark      { background: var(--color-ink); color: #fff; }
.ef-btn-dark:hover { background: var(--color-rose); }
.ef-btn-outline   { background: transparent; border: 1.5px solid var(--color-border-warm); color: var(--color-ink); }
.ef-btn-ghost-white { background: transparent; border: 1.5px solid rgba(255,255,255,.4); color: #fff; }
4

Navbar Unification

Replace hardcoded colours, standardise the CTA label

~1 hour Risk: Medium — test carefully
  • In Navbar.tsx, replace #3C3C3C → text-charcoal, #E4DDD7 → border-[var(--color-border-warm)], rgba(255,255,255,0.97) → bg-white/[0.97]
  • Change the navbar CTA label from "Contact Us" → "Book a Free Consultation" on both desktop and mobile menu
  • The homepage's own nav already says "Book Consultation" — update to match the exact same label: "Book a Free Consultation"
  • The homepage nav can remain as its own component (it has unique scroll-transparent behaviour specific to the split hero) — just ensure labels and tokens are aligned
  • Result: Consistent CTA across every page. Token-driven nav that respects the design system.
5

Inner Page Standardisation

Apply consistent patterns across all marketing pages

~3 hours Risk: Low
  • Section background rule — establish one rhythm and apply it to all inner pages: light (bg-background) → feature (bg-white) → warm accent (bg-warm) → dark (bg-ink) → CTA (<CTASection />)
  • Card consistency — replace all inline border border-[#E4DDD7] rounded-3xl patterns with the ef-card class. Currently inconsistent across About, Pricing, and FAQs pages.
  • Eyebrow → h2 → body hierarchy — standardise spacing across all sections: eyebrow mb-4, h2 mb-5, body mb-8 before any CTA. Currently varies between mb-4 and mb-5 on eyebrows inconsistently.
  • Replace inline button strings with ef-btn-primary / ef-btn-outline classes (from Phase 3)
  • Pages to touch: About, Pricing, Personal Training, FAQs, Contact, Blog, and legal pages (Privacy, Terms, Cookies)
  • Result: Every marketing page feels like it came from the same hand.
6

Hub Cohesion

Dynamic breadcrumb, token cleanup, minor polish

~2 hours Risk: Low
  • Dynamic breadcrumb — replace the hardcoded "Dashboard" in layout.tsx with a client component that reads usePathname() and maps to human-readable labels (Dashboard, Clients, Exercise Library, Medical Tracker, Agreements, Site Review)
  • HubSidebar SVG — replace hardcoded hsl(318, 86%, 63%) on the logo icon with var(--color-rose)
  • Hub colour tokens — audit the hub pages for any remaining hardcoded hex values and replace with the token system
  • Keep DM Sans for hub headings — appropriate for a data-dense back office; this is an intentional divergence
  • Keep shadcn Button — the rounded-full overrides on hub buttons already align them visually with the marketing site style
  • Result: Hub feels connected to the brand and is easier to navigate.

Preserve

What to Keep As-Is

Not everything needs changing. These elements are working well and should be kept:

Homepage scoped CSS (.efhome)

Correct pattern. Prevents style leakage. Keep the scope — just convert the variables.

Hub shadcn/ui components

Right choice for a data-dense back office. Don't replace with marketing-site patterns.

Rose/Teal/Cream palette

Well-chosen and distinctive for the brand. Consistent in intent if not execution.

DM Serif Display + DM Sans pairing

Strong, appropriate, and already loaded. Don't change the font choice.

Hub DM Sans headings

Correct for a data-dense back office. An intentional divergence from the marketing site.

Homepage split hero layout

A deliberate high-impact design distinct from the inner page full-bleed photo heroes. Keep both patterns.
